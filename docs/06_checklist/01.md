# Чек-лист по решению задач на экзамене 1С:Специалист

## Общее
* Включение управляемых блокировок
* создание подсистем, назначение картинок;
* создание подчиненных подсистем для документов, справочников и регистров;
* настройка интерфейса: разделы слева, открытые снизу

### Свойства и характеристики
*приведен пример для справочника Номенклатура, у вас может быть другой справочник*

* создать справочник **ЗначенияСвойств**, в качестве владельца указать ПВХ **СвойстваОбъектов**;
* в ПВХ **СвойстваОбъектов** указать типы: **Строка** и справочник **ЗначенияСвойств** (тип **Строка** добавляем по желанию для проверки работы отбора по типу);
* в ПВХ **СвойстваОбъектов** в качестве дополнительных значений указать справочник **ЗначенияСвойств**;
* создать РС **ЗначенияХарактеристикНоменклатуры**:
    * измерения: 
        * Характеристика (тип ПВХ СвойстваОбъектов);
        * Номенклатура (тип справочник Номенклатура);
    * ресурсы:
        * Значение (тип характеристика СвойстваОбъектов).
    * у всех измерений запрет незаполненных значений;
    * у измерения **Номенклатура** галка **Ведущее**;
    * у ресурса **Значение** связь по типу измерения **Характеристика**;
    * у ресурса **Значение** связь параметров выбора по владельцу измерения **Характеристика**;

## Валютный учет
* бухгалтерский учет ведется в национальной валюте (рубль);
* справочник Валюты и регистр сведений Курс валют уже есть в каркасной конфигурации;
* проведение операций в валюте по курсу на дату документа;
* если не известно сколько валют, то  измерение Валюта и 2 ресурса: Сумма и СуммаВал;
* если валюты перечислены, к примеру рубль, доллар и евро, тогда измерение не надо, ресурсы: Сумма, СуммаUSD, СуммаEUR$
* остатки по обеим суммам остатки должны выводиться в ноль:
    * списание курсовых разниц;
    * списание валюты пропорционально рублю (аналогично списанию себестоимости);

## Оперативный учет
* регистр накопления:
    * разрешить разделение итогов;
    * количество точность 0;
    * сумма точность 2;
    * измерения - запрет незаполненных значений;
* приходная накладная:
    * только пишем, не читаем - блокировка не нужна;
    * исключение дублей строк - группировка по номенклатуре;
    * исключение услуг из складского учета;
    * приходуем, поэтому контроль отрицательных остатков не нужен;
    * вывод движений в типовой форме;
    * удаление движений;
* расходная накладная (традиционная методика)
    * удаление движений (.Записать());
    * блокировка (из синтаксис-помощника);
    * чтение данных (запрос из документа и регистров);
    * контроль остатков (сообщить и отказ);
    * формирование движений (списание по партиям).
    * выход из обработки проведения с записью движений;
* расходная накладная (новая методика)
    * чтение данных (запрос);
    * формирование движений (запрос из документа);
    * запись движений с блокировкой;
    * контроль остатков запросом отрицательных;
    * выход из обработки проведения без записи движений;

## Отчеты
* все отчеты на СКД, за исключением диаграммы Ганта;
* в полях отчета только объекты, без реквизитов;
* в отличие от запросов в коде в отчетах не требуется заполнение параметров виртуальных таблиц;
* стандартный период;
* сводная таблица с помощью мастера;
* внешний вид отчета должен соответствовать:
    * заголовок отчета с помощью дополнительной группировки с вариантом "дополнительная информация" с макетом группировки;
    * шапка таблицы с помощью макета заголовка группировки;
    * шапка сводной таблицы с помощью макета группировки;
    * отображение параметров;
    * расположение итогов;
    * расположение колонок вертикально или горизонтально с помощью групп колонок;
    * заголовки полей.
