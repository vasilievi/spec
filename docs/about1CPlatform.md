# О платформе 1С

## Что такое платформа 1С
* основа для разработки и запуска любых бизнес-приложений;
* фреймворк, повышение уровня абстракции, работа с прикладными объектами;
* СУБД, работа с таблицами прикладных объектов;
* 2 режима работы: файл-сервер и клиент-сервер (MS SQL, PostgreSQL).

## Решаемые задачи
* задачи учета хозяйственной деятельности предприятий;
* для сдачи экзамена необходимы знания основ учета;
* каждую задачу нужно рассматривать с 2 сторон:
    * с точки зрения платформы;
    * с точки зрения пользователя;
* перед программированием нужно ответить на вопрос, смогу ли я решить на бумаге?

## Основы баз данных
* реляционная база данных - данные в связанных таблицах;
* таблица - ячейки, образованные пересечением строк и колонок;
* строка, запись, поле, столбец;
* элемент справочника, документ, строка ТЧ, реквизит;

## Что нужно знать о платформе 1С
* учёт ради отчёта:
    * проектирование базы данных;
    * запись в базу данных;
    * чтение из базы данных;
* CRUD - четыре базовые функции, используемые при работе с базами данных: создание, чтение, модификация, удаление. 
* проектирование базы данных осуществляется в конфигураторе **визаульно** в дереве конфигурации;
* элементы в дереве конфигурации называют объектами;
* платформа не работает напрямую с данными, она работает с СУБД;
![1c_sql](./img/1c_sql.png)
* 2 варианта обращения к данным: объектный (чтение и запись) и через запрос (только чтение);
* клиент-серверная архитектура, тонкий клиент, web-технологии - на экзамене знания не проверяются;
* интерактивная и программная работа с данными;
* работа с данными только в пользовательском режиме;
* механизмы платформы, программист,  пользователь;
* 2 типа объектов: ссылочные (есть стандартный реквизит Ссылка) и нет;
